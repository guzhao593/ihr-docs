  • 使用Mybatis Plus提供的条件构造器快速构建条件查询；
  • 使用Mybatis Plus提供的更新/插入策略进行数据操作；

  • 使用Mybatis Plus提供的分页插件；

其他功能一律不推荐大范围使用。

# 常用配置

# @MapperScan [Required]

@MapperScan("com.meicloud.**.mapper")
@SpringBootApplication
public class SaharaDemoApplication {}
1
2
3

添加@MapperScan，MapperScannerRegistrar会负责将扫描@MapperScan指定包路径下的Mapper接口注册成Bean，否则无法注入Mapper使用。

报错示例如下：

A component required a bean of type 'com.meicloud.**.mapper.***Mapper' that could not be found.
1

TIP

  • 一般而言，直接在应用启动类进行配置即可
  • 配置@MapperScan后，无需在Mapper接口上再添加@Mapper

# mapper-locations [Required]

指定Mapper接口所对应的新定义XML文件位置，Maven多模块项目的扫描路径需以 classpath*: 开头，参考示例如下：

mybatis-plus:
  mapper-locations: 'classpath*:META-INF/conf/mybatis/*Mapper.xml'
1
2

# type-aliases-package [Optional]

给包中的类注册别名，注册后在Mapper对应的XML文件中可以直接使用类名，而不用使用全限定的类名。通常用于给实体或常量类注册别名，以便可以直接在Mapper xml文件快速引用。配置示例如下：

mybatis-plus:
     type-aliases-package: 'com.meicloud.**.entity'
1
2

# type-handlers-package [Optional]

TypeHandler 扫描路径，如果配置了该属性，SqlSessionFactoryBean 会把该包下面的类注册为对应的自定义类型转换器TypeHandler。如果我们有特殊的java对象类型和JdbcType需要转换，可以添加相应的TypeHandler，再结合type-handlers-package配置加载自定义TypeHandler。

# map-underscore-to-camel-case

决定Java属性和数据库字段之间是否开启自动驼峰命名规则（camel case）映射。默认开启，不推荐进行修改，如需修改个别字段的映射规则请使用@TableField。

# auto-mapping-behavior

MyBatis自动映射策略，通过该配置可指定MyBatis是否并且如何来自动映射数据表字段与对象的属性，总共有 3 种可选值：

  • NONE：不启用自动映射

  • PARTIAL：只对非嵌套的 resultMap 进行自动映射（默认值）

  • FULL：对所有的 resultMap 都进行自动映射

    建议采用默认值即可

# auto-mapping-unknown-column-behavior

MyBatis 自动映射时未知列或未知属性处理策略，通过该配置可指定 MyBatis 在自动映射过程中遇到未知列或者未知属性时如何处理，总共有 3 种可选值：

  • NONE：不做任何处理 (默认值)

  • WARNING：以日志的形式打印相关警告信息

  • FAILING：当作映射失败处理，并抛出异常和详细信息

    建议开发过程使用FAILING，以便快速发现问题，线上环境使用WARNING或NONE即可

# db-config

  • id-type: 全局主键类型，默认ASSIGN_ID（com.baomidou.mybatisplus.annotation.IdType）
  • table-prefix: 表名前缀，默认为null
  • table-underline: 表名是否驼峰映射。默认为true
  • logic-delete-field: 逻辑删除字段配置
  • logic-delete-value: 逻辑删除值
  • logic-not-delete-value: 逻辑未删除值

# 常用注解

# @TableId

主键注解，作用于实体属性，通过idType指定主键生成策略，默认为ASSIGN_ID

值 描述
ASSIGN_ID 分布式ID，主键类型为Long和Integer，IdentifierGenerator#nextId
ASSIGN_UUID 分布式UUID,主键类型为String，IdentifierGenerator#nextUUID
AUTO 数据库ID自增
NONE/ INPUT 自行设置主键

# @TableField

字段注解(非主键)，常用属性如下：

  • value(): 指定映射数据库字段名称
  • exist(): 指定是否为数据库字段，默认为true，代表CRUD不会处理该字段。
  • insertStrategy(): 指定insert操作时，该字段拼接insert语句时的策略，策略如下：
    • IGNORED: 忽略判断，代表插入时不处理该字段,该是怎么样就怎么样；
    • NOT_NULL: 非NULL判断，代表仅null字段不插入，即使为“”也插入；
    • NOT_EMPTY: 非空判断，null和“”都不插入；
    • DEFAULT: 追随全局策略，默认全局策略为NOT_NULL;
    • NEVER: 不加入SQL
  • updateStrategy(): 指定update操作时，该字段拼接update语句时的策略，策略参考insertStrategy()
  • fill(): 字段填充策略，策略如下:
    • DEFAULT 默认不处理
    • INSERT 插入时填充
    • UPDATE 更新时填充
    • INSERT_UPDATE 插入和更新时填充字
  • numericScale(): 指定小数点后保留的位数

# @Version

乐观锁注解，标记 @Verison 在字段上，即可实现乐观锁功能。

# @TableLogic

表字段逻辑处理注解（逻辑删除）

属性 类型 必须指定 默认值 描述
value String 否 "" 逻辑未删除值
delval String 否 "" 逻辑删除值

# AbstractWrapper

# allEq

全部相等(或个别isNull)

allEq(Map<R, V> params)
allEq(Map<R, V> params, boolean null2IsNull)
allEq(boolean condition, Map<R, V> params, boolean null2IsNull)
1
2
3
  • params: key为数据库字段名，value为字段值
  • null2IsNull: 是否调用isNull()进行null判断
    • true 空判断，存在value为null，则拼接is null，默认为true;
    • false 忽略空判断，存在value为null，则不拼接is null;
  • condition: 执行条件，判断是否需要拼接该条件，默认为true，需要拼接；

TIP

condition条件可用于处理非空判断等

示例：

Map<String, Object> paramMap = new HashMap<>();
paramMap.put("employee_name", "老宋");
paramMap.put("age", null);

QueryWrapper<Map<String, Object>> queryWrapper = new QueryWrapper<>();
// employee_name = '老宋' and age is null
queryWrapper.allEq(paramMap);
// employee_name = '老宋'
queryWrapper.allEq(paramMap, false);

# 常见问题

# selectByMap/deleteByMap无法使用驼峰字段进行查询

xxxByMap底层默认直接使用数据库字段构造查询，Sahara 2.0对底层进行了重写支持，直接使用即可；

# 条件构造器Wraper无法使用驼峰字段进行查询

Wraper底层默认采用数据库字段构造查询条件，Sahara 2.0对底层进行了重写支持，使用SaharaQWrapper/SaharaUWrapper替代QueryWrapper/UpdateWrapper。

# 使用数据库关键字作为属性数据查询/操作报错

@TableField(value = "`status`")
private Boolean status;
1
2

# ID_WORKER生成主键太长导致 js 精度丢失

JavaScript 无法处理 Java 的长整型 Long 导致精度丢失，具体表现为主键最后两位永远为 0，sahara框架对Jackson序列化框架添加了Long转String的序列化器，添加相应配置即可（详见sahara）

# insert 后如何返回主键

insert 后主键会自动 set 到实体的 ID 字段，所以你只需要 getId() 就好

# 如何排除非表中字段

@TableField(exist=false)
private String noColumn;

# Caused by: org.apache.ibatis.binding.BindingException: Invalid bound statement (not found)

检查mybatis-plus.mapper-locations的配置，是否自定义SQL不在扫描范围内；
