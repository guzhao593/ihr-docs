# 动态数据源组件

该组件主要用于支持多数据源读写

# 功能特性

  • 支持读写分离（读库对称场景）；
  • 支持不同数据源读写（主数据场景）；
  • 内置支持druid、dbcp2连接池，支持多种连接池扩展；
  • 内置支持轮询、随机、权重数据源选择策略，支持自定义数据源选择策略；
  • 支持多线程场景使用相同数据源；

# 使用说明

  • 多数据源方案的注解原则上支持注解在所有的bean及其方法上，但是建议注解到Service相应的方法上；
  • 多数据方案的注解支持数据源嵌套，但是需要注意不能使用多事务，因为暂时不考虑引入分布式事务，暂时无法解决多数据源事务嵌套问题；

# 业务读写注解

  • ReadOperation 标识业务为读操作，后续从读源根据数据源选择策略决定数据源
    • dataSourceCode 指定数据源，读写分离场景下不需要使用
  • WriteOperation 标识业务为写操作，后续从读源根据数据源选择策略决定数据源
    • dataSourceCode 指定数据源，读写分离场景下不需要使用

# 动态数据源

  • DynamicDataSource 动态数据源抽象类
  • DynamicDbcp2DataSource 重写DynamicDataSource#createDataSource创建dbcp2数据源
  • DynamicDruidDataSource 重写DynamicDataSource#createDataSource创建druid数据源

# 数据源选择策略

  • PollingDataSourceStrategy 轮询策略，默认策略
  • RandomDataSourceStrategy 随机策略
  • WeightDataSourceStrategy 权重策略

# 自定义动态数据源

参考dbcp2实现

@Configuration
@ConditionalOnProperty(prefix = "spring.datasource.dynamic", name = "active", havingValue = "true")
@ConditionalOnClass(BasicDataSource.class)
@AutoConfigureAfter(DynamicDataSourceAutoConfiguration.class)
@AutoConfigureBefore({DataSourceAutoConfiguration.class, SaharaMybatisPlusAutoConfiguration.class, MybatisPlusAutoConfiguration.class})
@EnableConfigurationProperties({DynamicDbcp2DataSourceAutoConfiguration.Dbcp2DataSourceProperties.class, DynamicDataSourceProperties.class})
public class DynamicDbcp2DataSourceAutoConfiguration {

    @Bean
    @ConditionalOnMissingBean
    public DataSource dataSource(DynamicDataSourceProperties dynamicDataSourceProperties, DataSourceStrategy dataSourceStrategy, Dbcp2DataSourceProperties properties) {
        DynamicDbcp2DataSource dataSource = new DynamicDbcp2DataSource();
        dataSource.setDynamicDataSourceProperties(dynamicDataSourceProperties);
        dataSource.setDataSourceStrategy(dataSourceStrategy);
        dataSource.setBasicDataSource(properties.getDbcp2());
        return dataSource;
    }

    @Data
    @ConditionalOnClass(BasicDataSource.class)
    @ConfigurationProperties("spring.datasource")
    static class Dbcp2DataSourceProperties {
        private BasicDataSource dbcp2;

    }
}

# 自定义数据源选择策略

  • 实现DataSourceStrategy，重写determine()方法

    public class PollingDataSourceStrategy implements DataSourceStrategy {

    private final AtomicInteger counter = new AtomicInteger(0);

    @Override
    public String determine(List<SaharaDataSource> dataSources) {
        int index = Math.abs(counter.getAndAdd(1) % dataSources.size());
        SaharaDataSource saharaDataSource = dataSources.get(index);
        return saharaDataSource.getDataSourceCode();
    }
}

# 常见问题

# 为什么不直接使用dynamic-datasource-spring-boot-starter

  1. dynamic-datasource-spring-boot-starter虽然支持了多种数据源，但是都是从别人的starter那边直接拷贝过来的，设计上不是很好，而且后期不好维护；
  2. 读写分离我们更希望只需要业务指定读写操作即可，而不是将具体数据源指定到代码中，即使它支持规则匹配；
  3. 多数据源方案的实现成本不高，我们针对内部场景可以自己设计实现；
