    # 功能权限

    i-toolbar 提供了 filterTool 方法用于过滤工具集合，但实际上在业务开发过程中，我们更多的是直接操作数组 tools 的方式实现的。那么这个方法是否能够结合「功能权限」使用呢？

    # 问题

    在过去，我们要实现某个按钮的显示隐藏，需要全局或者逐个页面注入「权限 mixin」，再在每个按钮上面绑定 key 之余，还要写上 v-if 和权限变量才能实现。虽然能实现，但显而易见，这种方式算不上好：

    1. 不仅需要 v-if 绑定对应的权限变量，还得跟业务原有的显隐逻辑冗余（类似于 v-if="权限变量 && (业务变量A || 业务变量B) && 业务变量C && ..."）
    2. 系统按钮千千万，表格表单占一半。也就意味着要实现同样的逻辑，需要把「权限 mixin」中方法和变量散落得到处都是，再由开发者手动在业务页面中调用

    # 方案

    首先 i-toolbar 已经集成了 Element 的按钮、按钮集合、下拉菜单、弹出框，支持自定义渲染，有能力替代项目中单独使用如上组件的场景。并且本身就是数据驱动的，拥有 filterTool 方法执行过滤，那么是否可以直接在全局定义此方法，实现同样的过滤逻辑呢？答案是肯定的。

    1. 开发者在绑定 key 后只需要关心按钮在业务中显隐逻辑即可
    2. 表格表单等一系列 Sahara 组件的工具区都内置了 i-toolbar，因此该方案能控制所有组件中的按钮显隐，一劳永逸

    # 实现

    那么实际上我们就是要解决「如何将 filterTool 的默认行为更改为：根据全局权限状态和 key 执行过滤」。可以在注册组件之前，通过如下方式改写组件属性的默认值。

    // 覆盖原始默认值
Sahara.Toolbar.props.filterTool.default = filterTool

    2

    由于存在按钮集合、下拉菜单的情况，因此过滤函数需要考虑递归逻辑。实现如下：

    import Vue from 'vue'
import Sahara from '@ipeople/sahara'

// 请求权限信息至全局状态
const globalAccessKeys = fetchAccessKeys()

const filterTool = function (tool) {
  // 过滤绑定了 key 但不存在于全局权限 keys 的工具
  if (tool.key && globalAccessKeys.indexOf(tool.key) === -1) {
    return false
  }
  return true
}

// 更改 i-toolbar filterTool 属性的默认值
Sahara.Toolbar.props.filterTool.default = filterTool

Vue.use(Sahara)
