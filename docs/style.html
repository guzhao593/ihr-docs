# 样式

# 预处理器

推荐使用 Sass, 目前世面上一些主流的 UI 框架使用的都是 Sass 作为 CSS 的预处理器，所以这里推荐使用 Sass

# 目录结构

推荐全局样式都放在 @/src/styles 目录下

├── styles
│   ├── reset.scss               # 重置样式
│   ├── common.scss              # 通用样式
│   ├── btn.scss                 # 按钮样式
│   ├── sidebar.scss             # sidebar css
│   ├── function.scss            # scss函数
│   ├── element-ui.scss          # 全局自定义 element-ui 样式
│   ├── index.scss               # 全局通用样式
│   ├── mixin.scss               # 全局mixin
│   ├── transition.scss          # vue transition 动画
│   └── variables.scss           # 全局变量

# 组件和单独页面样式

一般情况下，全局样式都写在 @/src/styles 目录下，每个组件可以先使用全局样式，但页面自己对应的样式都写在自己的 .vue 文件之中

且按照 风格指南，组件自己的样式一定要在 style 标签加上 scoped 属性

# 自定义 element-ui 样式

现在我们来说说怎么覆盖 element-ui 样式。由于 element-ui 的样式我们是在全局引入的，所以你想在某个页面里面覆盖它的样式就不能加 scoped，但你又想只覆盖这个页面的 element 样式，你就可在它的父级加一个 class，用命名空间来解决问题。

.article-page {
  /* 你的命名空间 */
  .el-tag {
    /* element-ui 元素*/
    margin-right: 0px;
  }
}

# 深度选择器

当你子组件使用了 scoped 但在父组件又想修改子组件的样式可以 通过 >>> 或者 /deep/ 来实现：

<style scoped>
.content >>> .el-input { /* ... */ }
</style>
1
2
3

将会编译成

.content[data-v-f3f3eg9] .el-input {
  /* ... */
}
1
2
3

有些像 Sass 之类的预处理器无法正确解析 >>>。这种情况下你可以使用 /deep/ 或 ::v-deep 操作符取而代之——两者都是 >>> 的别名，同样可以正常工作。

例如你想覆盖某个特定页面 element 的 button 的样式，你可以这样做：

<style scoped>
.container {
  /deep/ .el-button { /* ... */ }
}
</style>

# Postcss

这里再来说一下 postcss 的配置问题，官方文档

跟 webpack 搭配使用，在 webpack.config.js 中使用 postcss-loader

module.exports = {
  module: {
    rules: [
      {
        test: /\.css$/,
        exclude: /node_modules/,
        use: [
          {
            loader: 'style-loader',
          },
          {
            loader: 'css-loader',
            options: {
              importLoaders: 1,
            }
          },
          {
            loader: 'postcss-loader'
          }
        ]
      }
    ]
  }
}

如上面代码所述的，autoprefixer 会去读取 package.json 下 browserslist 的配置参数

  • > 1% 兼容全球使用率大于 1%的浏览器
  • last 2 versions 兼容每个浏览器的最近两个版本
  • not ie <= 10 不兼容 ie8 及以下
