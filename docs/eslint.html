# ESlint

不管是多人合作还是个人项目，代码规范都是很重要的。这样做不仅可以很大程度地避免基本语法错误，也保证了代码的可读性。

# 安装ESLint

npm i eslint eslint-loader babel-eslint -D
1

# VSCode 配置 ESLint

推荐 eslint + VSCode 来进行开发

每次保存，VSCode 就能标红不符合 eslint 规则的地方，同时还会做一些简单的自我修正。VSCode 中安装步骤如下：

在插件栏搜索并安装 eslint 插件

安装并配置完成 ESLint 后，我们继续回到 VSCode 进行扩展设置，依次点击 文件 > 首选项 > 设置 打开 VSCode 配置文件,添加如下配置

{
  "files.autoSave": "off",
  "eslint.validate": [
    "javascript",
    "javascriptreact",
    "vue-html",
    {
      "language": "vue",
      "autoFix": true
    }
  ],
  "eslint.run": "onSave",
  "eslint.autoFixOnSave": true
}

# 配置项

所有的配置文件都在 .eslintrc.js 中。 一般项目基本规范是依托于 standard 。(大家可以按照自己的需求进行定制化配置)

注意：配置的 rules 属性禁止个人单独修改，若要修改，需要在团队内沟通，大家一起决定

module.exports = {
  parserOptions: {
    ecmaVersion: 2018,      // use es9 syntax
    sourceType: 'module',   // use ESM syntax
    ecmaFeatures: {
      impliedStrict: true,  // enable global strict mode
      jsx: true             // enable JSX
    }
  },

  env: {
    browser: true,          // allow browser global variables
    node: true,             // allow Node.js global variables and Node.js scoping
    commonjs: true          // allow CommonJS global variables and CommonJS scoping (used by webpack)
  },

  extends: 'standard',      // 你可以修改为  extends: 'vue' // 或者 extends: 'airbnb', 等等

  rules: {                  // 可以自定义一些规则
    'arrow-parens': 0,      // allow parenthesis-free arrow function
  }
}
