# 构建和发布

# 构建

项目开发完毕，只需要运行一行命令就可以打包你的应用：

# 打包正式环境
npm run build
1
2

构建打包成功之后，会在根目录生成 dist 文件夹，里面就是构建打包好的文件，通常是 ***.js 、***.css、index.html 等静态文件。

如果需要自定义构建，比如指定 dist 目录等，则需要通过对应 config 文件的 output.path 进行配置。

# 环境变量

如果是 @vue/cli2 创建的项目，环境变量配置在 config 文件夹下的对应 dev.env.js，prod.env.js，test.env.js 文件中

这里以 dev.env.js 文件中的配置来举例：

'use strict'
const merge = require('webpack-merge')
const prodEnv = require('./prod.env')

module.exports = merge(prodEnv, {
  NODE_ENV: '"development"'
})

如果是 @vue/cli3 创建的项目，则配置在根目录下的 .env.xxxx 文件中，所有测试环境或者正式环境变量的配置都在 .env.development等 .env.xxxx文件中。

在文件中直接写

FOO=bar
VUE_APP_SECRET=secret
1
2

即可，详情可参照 Vue CLI 官网

它们都会通过 webpack.DefinePlugin 插件注入到全局。

你在代码中可以通过如下方式获取:

console.log(process.env.VUE_APP_xxxx)
