# 风格指南

该风格指南主要是参照 vue 官方的风格指南。建议先阅读一遍指南，这能帮助让你写出更规范和统一的代码。当然每个团队都会有所区别。其中大部分规则也都配置在了eslint-plugin-vue之中，当没有遵循规则的时候会报错，详细内容见eslint章节。

当然也有一些特殊的规范，是不能被 eslint 校验的。需要人为的自己注意，并且来遵循。最主要的就是文件/文件夹的命名规则。

# 命名规则

# 项目命名

【强制】全部使用小写字母，单词间以中划线分隔

例：core-hr-fe、mengine-ui、jiebao-plus 等

# 目录命名

【强制】全部使用小写字母，单词间以中划线分隔

例：env-config、layout-pro、air-order 等

# 文件命名

包含 .js、 .json、 .vue、 .md、 .css、 .scss、 .less、 .img 等文件

【强制】全部使用小写字母，单词间以中划线分隔

例：native-modules.js、package-lock.json、i-tree-component.vue、saraha-extends.scss、close-icon.img 等

以上可以总结：项目、文件类都以 (kebab-case) 短横线 来命名，这也是参考各大厂/开源代码仓库的风格

# 变量的命名

【强制】变量使用 (camelCase) 驼峰命名法

例：

let currentDate = new Date()
1

# 常量的命名

【强制】常量使用全部字母大写，单词间以 (snakeCase)下划线分隔 的命名法

例：

const LOADING_TEXT = ''
1

# 函数的命名 及 函数参数的命名

【强制】函数和函数的参数都使用(camelCase)驼峰命名法，且函数的命名遵循动宾短语

例：

function formatString (sourceString) {
  return ...
}

【强制】boolean类型的变量使用is或has开头

例：

【强制】class 使用(kebab-case)短横线命名法

# CSS/SCSS 使用规范

# 缩进

【强制】采用2个空格

# 分号

【强制】每个属性声明末尾都要加分号

# 空格

【强制】类名后要加上空隔

# 注释

【强制】注释统一使用用'/* */'

# 命名

【强制】scss中的变量、函数、混合采用驼峰命名法

# 属性声明顺序

【强制】

  1. 显示属性：flex/ display/ float / overflow/ position / top / right / bottom / left 等

  2. 自身属性：border / margin / padding / width / height 等

  3. 描述属性：font / line-height / text-align / word-wrap/background / color / transition / list-style等

例：

.ration-order {
  display: block;
  float: right;
  position: absolute;
  top: 0;
  right: 0;
  bottom: 0;
  left: 0;
  z-index: 100;

  border: 1px solid #e5e5e5;
  border-radius: 3px;
  width: 100px;
  height: 100px;

  font: normal 13px "Helvetica Neue", sans-serif;
  line-height: 1.5;
  text-align: center;
  color: #333;
  background-color: #f5f5f5;
  opacity: 1;
}

# 模块引入

【强制】异步加载模块使用ES7标准的 () => import(/* webpackChunkName: "system" */'@/views/system/log')，使用模块名字合理划分异步加载模块

# 组件开发规范

【强制】每个组件必须定义name属性，采用帕斯卡命名法

【强制】props属性声明必须定义类型

【强制】通用组件需要根据组件文件名以帕斯卡命名法定义name属性

【强制】需要配有README使用文档

【强制】不允许在通用组件中定义全局变量，可定义全局常量

# 组件中的属性书写顺序

  v-for

  v-pre
  v-once

  ref
  key
  slot

  v-model

5.其他特性
  props

6.事件
  v-on

7.书写方式
  v-bind:attribute统一使用:attribute
  v-on:event统一使用@event
  字面量属性排在:attribute属性之后
  Boolean类型属性如果为true直接写上属性名即可

# 组件注册

【强制】通用组件全局注册，其他页面中不得进行再次注册

【强制】局部组件局部注册，不允许全局注册

# 组件引入外部资源

【强制】单文件组件style中不允许使用@import .scss文件，这样会造成打包重

# 组件样式

【强制】对于本组件中使用的样式，必须在 style 标签中添加 scoped 属性

# 标签书写规范

【强制】必须使用双闭合标签

【强制】标签内超过三个属性换行，一个属性一行，开标前右尖括号换行，如果没内容，闭标签紧跟开标签，如：

<i-select-tree
   v-model="searchParam.orgUnitId"
   :multiple="false"
   dataKey="org"
   nodeKey="orgUnitId"
   size="small"
   lazy
></i-select-tree>
如果有内容，闭标签换行
<el-form-item label="调出时间起">
   <el-date-picker
     v-model="searchParam.effectiveDateStart"
     :picker-options="effectStartOptions"
     type="date"
     @change="handleEffectiveDateStartChange"
></el-date-picker>
</el-form-item>

# 使用规范

# 变量

【强制】不允许使用var定义变量，定义常量用const，变量用let

【强制】不允许未经商讨随意定义全局变量

# 条件

【强制】 在 Equality Expression中使用类型严格的 ===。仅当判断 null 或 undefined 时，允许使用 ==。

解释： 使用 === 可以避免等于判断中隐式的类型转换。

例：

// good
if (age === 30) {
    // ......
}

// bad
if (age == 30) {
    // ......
}

# 循环

【强制】不要在循环体中包含函数表达式，事先将函数提取到循环体外。

解释：循环体中的函数表达式，运行过程中会生成循环次数个函数对象。

例：

// good
function clicker() {
    // ......
}

for (let i = 0, len = elements.length; i < len; i++) {
    let element = elements[i]
    addListener(element, 'click', clicker)
}
// bad
for (let i = 0, len = elements.length; i < len; i++) {
    let element = elements[i]
    addListener(element, 'click', function () {})
}

【强制】对有序集合进行遍历时，缓存 length。

解释：虽然现代浏览器都对数组长度进行了缓存，但对于一些宿主对象和老旧浏览器的数组对象，在每次 length 访问时会动态计算元素个数，此时缓存 length 能有效提高程序性能。

示例：

for (let i = 0, len = elements.length; i < len; i++) {
    let element = elements[i]
    // ......
}

# 注释

# 单行注释

【强制】必须独占一行。// 后跟一个空格，缩进与下一行被注释说明的代码一致

# 文档化注释

【建议】 为了便于代码阅读和自文档化，以下内容必须包含以 /** ...*/ 形式的块注释中

  文件
  类
  函数或方法
  全局变量
  常量

【强制】文档注释前必须空一行

# 函数或方法注释

【强制】函数/方法注释必须包含函数说明，有参数和返回值时必须使用注释标识

【建议】当 return 关键字仅作退出函数/方法使用时，无须对返回值作注释标识

【强制】参数和返回值注释必须包含类型信息，且不允许省略参数的说明

【建议】当函数是内部函数，外部不可访问时，可以使用 @inner标识

例：

/**
 * 函数描述
 *
 * @param {string} p1 参数1的说明
 * @param {string} p2 参数2的说明，比较长
 *     那就换行了
 *     (是否需要换行要看规定的最短换行字数, 一般为换行字数为80, 
 *     在vscode的setting文件中的 属性 editor.rulers 定义).
 * @param {number=} p3 参数3的说明（可选）
 * @return {Object} 返回值描述
*/
function foo (p1, p2, p3) {
  let p3 = p3 || 10;
  return {
    p1: p1,
    p2: p2,
    p3: p3
  }
}

# 细节注释

【强制】有时我们会使用一些特殊标记进行说明。特殊标记必须使用单行注释的形式。下面列举了一些常用标记：

1.	TODO: 有功能待实现。此时需要对将要实现的功能进行简单说明。
2.	FIXME: 该处代码运行没问题，但可能由于时间赶或者其他原因，需要修正。此时需要对如何修正进行简单说明。
3.	HACK: 为修正某些问题而写的不太好或者使用了某些诡异手段的代码。此时需要对思路或诡异手段进行描述。
4.	XXX: 该处存在陷阱。此时需要对陷阱进行描述。
