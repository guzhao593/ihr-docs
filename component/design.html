# 扁平化参数

像 HTML 原生元素那样，只接受原始类型（字符串、数值、布尔值和函数）作为属性，避免复杂的对象。当然，数据除外。

<!-- good -->
<my-component
  label="hello"
  :actived="true"
  :width="600"
  show
></my-component>
<!-- bad -->
<my-component :config="myConfig"></my-component>

# 良好的接口设计

  1. 接口和已知概念保持一致。
  2. 如果常量变为 props 能应对更多的场景，那么就可以作为 props。原有的常量可作为默认值。
  3. 如果接口对应的功能点过大，可以考虑通过扩展等方式构建一个新的组件。
  4. 保证组件的属性和事件足够满足大多数使用场景。
