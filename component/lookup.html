# Lookup 通用选择器

作为业务选择器的基版。

可通过下拉选项执行简单搜索，也能以对话框的形式承载业务视图的复杂搜索，保持业务组件的一致性。

# 基本使用

使用方式跟常规的表单选择器一致。至于 business 参数可以具体查看全局配置。

# 全局配置

业务选择器的配置相对复杂且固定，适合集中管理。可以在 $SAHARA.lookup.views 中配置业务信息，便能在业务页面中通过 bussines 参数引用相关配置。

比如此处 views 下的配置信息名为 test，那么 business="test" 就能合并此处的配置信息至 Lookup 组件：




 
 
 
 
 
 
 
 
 





Vue.use(Sahara, {
  lookup: {
    views: {
      test: {
        title: 'test',                  // 对话框标题
        component: Test,                // 业务视图
        valueKey: '',                   // 对象数据唯一标识符，默认为 'value'
        props: {},                      // 数据映射
        querySearch (query) {},         // 搜索请求
        echo (values) {},               // 回显请求，初始时执行
        dialogProps: {},                // 对话框配置
      },
      // add more view config...
    }
  }
})

# 对象类型

如果绑定值为对象类型，在设置 value-type="object" 之余，请务必指定 value-key 作为它的唯一性标识，也可以一劳永逸将其写在全局配置中。

# 数据回显

当 value-type="object" 则直接以 value 中的对象作为回显数据来源。否则会执行 echo 方法获取数据，注意需要返回 Promise 对象。

# 视图类型

指定 type 为 view 时，可以直接使用其业务视图，默认是 dialog，即通过弹窗展示业务视图。

# 数据确认

在业务视图中的数据流回 Lookup 组件前，会触发 before-writeback 函数，用于对返回数据做确认处理。

TIP

  • 当 type 是默认的 dialog 时，before-writeback 在点击确定时触发
  • 当 type 是 view 时，before-writeback 是否触发取决于业务视图的 $emit('input') 事件，从而实现双向数据绑定
  • 获取 Lookup 实例执行 writeback 方法也能达成触发条件
